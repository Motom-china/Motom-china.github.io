##+(instancetype)modelWithDictionary(NSDictionary *)dictionary

终于到了关键一步，这里首先进行类型的判断是否为空，再yymodel里包含了很多类型判断和空值判断，（看来一个框架要考虑健壮性，还有极强的容错性。）
类声明class cls = [self class]这里返回的是我们自定的类（也就是YYBook类),然后使用_YYModelMeta类,这个类是干嘛用的呢？我们先来看看这个类

###_YYModelMeta
@interface _YYModelMeta : NSObject {
@package
YYClassInfo _classInfo;
/// Key:mapped key and key path, Value:_YYModelPropertyMeta.
NSDictionary _mapper;
/// Array<_yymodelpropertymeta>, all property meta of this model.
NSArray _allPropertyMetas;
/// Array<_yymodelpropertymeta>, property meta which is mapped to a key path.
NSArray _keyPathPropertyMetas;
/// Array<_yymodelpropertymeta>, property meta which is mapped to multi keys.
NSArray *_multiKeysPropertyMetas;
/// The number of mapped key (and key path), same to _mapper.count.
NSUInteger _keyMappedCount;
/// Model class type.
YYEncodingNSType _nsType;

    BOOL _hasCustomWillTransformFromDictionary;
    BOOL _hasCustomTransformFromDictionary;
    BOOL _hasCustomTransformToDictionary;
    BOOL _hasCustomClassFromDictionary;
}
@end

先从参数说起

###YYClassInfo *classInfo
这个类是什么意思呢？来看一下

第一个cls 指向自己，第二个superCls指向自己的父类，第三个类metaCls指向自己的元类。（如果这里有问题，那么你需要了解到runtime,以及类的的内部结构。）
isMeta判断自己是否是元类，name为类的名称YYClassInfo是对自己的父类的一个封装，剩下的三个NSDictionary :ivarInfos,methodInfos,propertyInfos是对类中属性(property)，方法(method)，ivar(class)的保存。

###其他参数

_mapper:存储 Json key 与 _PropertyMeta对象 的映射关系字典
_allPropertyMetas:数组存放所有的 PropertyMeta
_keyPathPropertyMetas:数组存放所有映射json keyPath 的 _PropertyMeta对象
_multiKeysPropertyMetas：数组存放所有映射多个json key 的 _PropertyMeta对象（一对多）

NSUInteger _keyMappedCount; 自定义类的属性个数（比如YYBook里面有三个属性，如果自定义类中包含另外的类B的话，类B以及其中的属性不算入其中）
_nsType：解析Class是Foundation类型哪一种
_hasCustomWillTransformFromDictionary、_hasCustomTransformFromDictionary、_hasCustomTransformToDictionary、_hasCustomClassFromDictionary，作用：判断YYModel协议方法是否实现
我们先根据流程 遇到哪个参数再讲。

##+(instancetype)modelWithDictionary(NSDictionary *)dictionary
先回到我们的步骤中去

1464行 对一个_YYModelMeta类进行了处理，那么我们跳进去看一下metaWithClass这个方法

##+ (instancetype)metaWithClass:(Class)cls

这一个方法开始涉及到底层的一些代码，其中有cf框架，它是一组c语言接口，通过调用底层的c可以使得代码的运行效率大大提高。( YYKit的效率高于现在其他的主流框架。效率对比请看这里)。开始仍然是是否为空的判断，然后出现了一个以CF开头的CFMutableDictionaryRef的声明，dispatch_once_t 线程的一个单例,dispatch_semaphore_t 线程的锁(这里是为了防止多线程造成的数据错误，也就是防止多个线程同时改变同一数据)。
dispatch_once(GCD，该函数只执行一次，之后便不在执行，想深入了解请看这里)。
cache（缓存）顾名思义 是用来存放数据的，之后的方法CFDictionaryCreateMutable方法为cache申请了一块内存空间用来存放类的相关数据。lock解锁。（想了解CFMutableDictionaryRef的同学请看这里）
因为第一次调用metaWithClass:(class)cls的时候cls转换meta为nil，所以我们进入到644行meta = [[_YYModelMeta alloc] initWithClass:cls];
