这里调用了一个_yy_dictionaryWithJSON方法进行json－nsdictionary的转换。点进去看一看

+(NSDictionary *)_yy_dictionaryWithJSON:(id)json

+ (NSDictionary *)_yy_dictionaryWithJSON:(id)json {
    if (!json || json == (id)kCFNull) return nil;
    NSDictionary *dic = nil;
    NSData *jsonData = nil;
    if ([json isKindOfClass:[NSDictionary class]]) {
        dic = json;
    } else if ([json isKindOfClass:[NSString class]]) {
        jsonData = [(NSString *)json dataUsingEncoding : NSUTF8StringEncoding];
    } else if ([json isKindOfClass:[NSData class]]) {
        jsonData = json;
    }
    if (jsonData) {
        dic = [NSJSONSerialization JSONObjectWithData:jsonData options:kNilOptions error:NULL];
        if (![dic isKindOfClass:[NSDictionary class]]) dic = nil;
    }
    return dic;
}

第一行判断 json是否为空，kCFNull 是 NSNull 的单例,具体的介绍请看本文。然后声明了NSDictionary和NSData两个类， 然后进行类型判断，使用oc自带的NSJSONSerialization来进行数据的转换，返回一个NSDictionary类型。

+(instancetype)modelWithDictionary(NSDictionary *)dictionary

+ (instancetype)modelWithDictionary:(NSDictionary *)dictionary {
    if (!dictionary || dictionary == (id)kCFNull) return nil;
    if (![dictionary isKindOfClass:[NSDictionary class]]) return nil;

    Class cls = [self class];
    _YYModelMeta *modelMeta = [_YYModelMeta metaWithClass:cls];
    if (modelMeta->_hasCustomClassFromDictionary) {
        cls = [cls modelCustomClassForDictionary:dictionary] ?: cls;
    }

    NSObject *one = [cls new];
    if ([one modelSetWithDictionary:dictionary]) return one;
    return nil;
}

终于到了关键一步，这里首先进行类型的判断是否为空，再yymodel里包含了很多类型判断和空值判断，（看来一个框架要考虑健壮性，还有极强的容错性。）
类声明class cls = [self class]这里返回的是我们自定的类（也就是YYBook类),然后使用_YYModelMeta类,这个类是干嘛用的呢？我们先来看看这个类

_YYModelMeta

@interface _YYModelMeta : NSObject {
    @package
    YYClassInfo *_classInfo;
    /// Key:mapped key and key path, Value:_YYModelPropertyMeta.
    NSDictionary *_mapper;
    /// Array<_YYModelPropertyMeta>, all property meta of this model.
    NSArray *_allPropertyMetas;
    /// Array<_YYModelPropertyMeta>, property meta which is mapped to a key path.
    NSArray *_keyPathPropertyMetas;
    /// Array<_YYModelPropertyMeta>, property meta which is mapped to multi keys.
    NSArray *_multiKeysPropertyMetas;
    /// The number of mapped key (and key path), same to _mapper.count.
    NSUInteger _keyMappedCount;
    /// Model class type.
    YYEncodingNSType _nsType;

    BOOL _hasCustomWillTransformFromDictionary;
    BOOL _hasCustomTransformFromDictionary;
    BOOL _hasCustomTransformToDictionary;
    BOOL _hasCustomClassFromDictionary;
}
@end

先从参数说起

YYClassInfo *classInfo

这个类是什么意思呢？来看一下

@interface YYClassInfo : NSObject
@property (nonatomic, assign, readonly) Class cls; ///< class object
@property (nullable, nonatomic, assign, readonly) Class superCls; ///< super class object
@property (nullable, nonatomic, assign, readonly) Class metaCls;  ///< class's meta class object
@property (nonatomic, readonly) BOOL isMeta; ///< whether this class is meta class
@property (nonatomic, strong, readonly) NSString *name; ///< class name
@property (nullable, nonatomic, strong, readonly) YYClassInfo *superClassInfo; ///< super class's class info
@property (nullable, nonatomic, strong, readonly) NSDictionary<NSString *, YYClassIvarInfo *> *ivarInfos; ///< ivars
@property (nullable, nonatomic, strong, readonly) NSDictionary<NSString *, YYClassMethodInfo *> *methodInfos; ///< methods
@property (nullable, nonatomic, strong, readonly) NSDictionary<NSString *, YYClassPropertyInfo *> *propertyInfos; ///< properties
@end

第一个cls 指向自己，第二个superCls指向自己的父类，第三个类metaCls指向自己的元类。（如果这里有问题，那么你需要了解到runtime,以及类的的内部结构。）这里是元类（metaClass）和Class的关系，详细的类介绍请参考这里

isMeta判断自己是否是元类，name为类的名称，YYClassInfo是对自己的父类的一个封装，剩下的三个NSDictionary :ivarInfos,methodInfos,propertyInfos是对类中属性(property)，方法(method)，ivar(class)的保存。

其他参数

_mapper:存储 Json key 与 _PropertyMeta对象 的映射关系字典
_allPropertyMetas:数组存放所有的 PropertyMeta
_keyPathPropertyMetas:数组存放所有映射json keyPath 的 _PropertyMeta对象
_multiKeysPropertyMetas：数组存放所有映射多个json key 的 _PropertyMeta对象（一对多）

NSUInteger _keyMappedCount; 自定义类的属性个数（比如YYBook里面有三个属性，如果自定义类中包含另外的类B的话，类B以及其中的属性不算入其中）
_nsType：解析Class是Foundation类型哪一种
_hasCustomWillTransformFromDictionary、_hasCustomTransformFromDictionary、_hasCustomTransformToDictionary、_hasCustomClassFromDictionary，作用：判断YYModel协议方法是否实现
我们先根据流程 遇到哪个参数再讲。

_YYModelMeta *modelMeta = [_YYModelMeta metaWithClass:cls], 对一个_YYModelMeta类进行了处理，那么我们跳进去看一下metaWithClass这个方法.

+ (instancetype)metaWithClass:(Class)cls

+ (instancetype)metaWithClass:(Class)cls {
    if (!cls) return nil;
    static CFMutableDictionaryRef cache;
    static dispatch_once_t onceToken;
    static dispatch_semaphore_t lock;
    dispatch_once(&onceToken, ^{
        cache = CFDictionaryCreateMutable(CFAllocatorGetDefault(), 0, &kCFTypeDictionaryKeyCallBacks, &kCFTypeDictionaryValueCallBacks);
        lock = dispatch_semaphore_create(1);
    });
    dispatch_semaphore_wait(lock, DISPATCH_TIME_FOREVER);
    _YYModelMeta *meta = CFDictionaryGetValue(cache, (__bridge const void *)(cls));
   NSDictionary *obj = CFDictionaryGetValue(cache, (__bridge const void *)(cls));
    NSLog(@"obj -- %@",obj);
    NSLog(@"cache - %@",cache);
    dispatch_semaphore_signal(lock);
    if (!meta ) {
        meta = [[_YYModelMeta alloc] initWithClass:cls];
        if (meta) {
            dispatch_semaphore_wait(lock, DISPATCH_TIME_FOREVER);
            CFDictionarySetValue(cache, (__bridge const void *)(cls), (__bridge const void *)(meta));
            dispatch_semaphore_signal(lock);
        }
    }
    return meta;
}

这一个方法开始涉及到底层的一些代码，其中有cf框架，它是一组c语言接口，通过调用底层的c可以使得代码的运行效率大大提高。( YYKit的效率高于现在其他的主流框架。效率对比请看这里)。开始仍然是是否为空的判断，然后出现了一个以CF开头的CFMutableDictionaryRef的声明，dispatch_once_t 线程的一个单例,dispatch_semaphore_t 线程的锁(这里是为了防止多线程造成的数据错误，也就是防止多个线程同时改变同一数据)。
dispatch_once(GCD，该函数只执行一次，之后便不在执行，想深入了解请看这里)。
cache（缓存）顾名思义 是用来存放数据的，之后的方法CFDictionaryCreateMutable方法为cache申请了一块内存空间用来存放类的相关数据。lock解锁。（想了解CFMutableDictionaryRef的同学请看这里）
因为第一次调用metaWithClass:(class)cls的时候cls转换meta为nil，所以我们进入到644行meta = [[_YYModelMeta alloc] initWithClass:cls];
